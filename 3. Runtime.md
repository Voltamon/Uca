# Runtime: Uca Execution & Deployment Architecture

The Uca runtime is engineered for hermetic development and resilient production orchestration. It integrates four distinct foundational frameworks—**PocketBase** (Go), **Preact** (TS), **PicoCSS** (TS), and **SmolAgents** (Python)—into a unified, high-performance execution unit.

---

## 1. Process Architecture: Supervision & Fault Tolerance

In both development and production environments, the application is anchored by a persistent **Supervisory Process** managed by the primary Golang binary.

### Supervision & Monitoring
- **Process Orchestrator**: The Go binary acts as the system supervisor, initializing the persistence layer (PocketBase) and orchestrating the AI execution environment.
- **Agent Watchdog**: The supervisor includes specialized telemetry logic to **continuously monitor the health of the Python agent process**.
- **Automated Recovery**: Should the Python runtime terminate unexpectedly or hang, the supervisor triggers an automated restart sequence to ensure constant availability of AI-driven services.

### Inter-Process Communication (IPC)
Component interoperability within the Uca stack is facilitated via a **Localized HTTP Bus** bound exclusively to `localhost`. This ensures a secure, low-latency communication channel that is isolated from external network interfaces.

---

## 2. Polyglot Interop Layer: The Unified Registry

The `.uca/` directory hosts the auto-generated interop logic that bridges the three discrete runtimes (Go, TS, Python) via a central **Unified Registry Matrix**.

- **Deterministic Mapping**: The Registry maintains an internal resolution map, ensuring that `pages: loader:` and `agents: tools:` definitions correctly reference their corresponding implementations in the `services/` layer.
- **Static Type Parity**: The runtime environment ensures end-to-end integrity by automatically mapping **Golang structures to TypeScript interfaces**, enabling seamless data transfer between the logic and presentation layers.
- **Automated Tool Export**: Service methods containing valid `desc:` metadata are automatically registered as authorized tools within the `smolagents` framework.

---

## 3. Iterative Development: Real-Time State Reconciliation

The `uca dev` command provisions a premium development environment characterized by multi-layer hot reloading and automated state synchronization.

- **Hot Module Reloading (HMR)**: The runtime provides sub-second reconciliation for changes detected in the `pages/`, `services/`, and `agents/` directories.
- **State Preservation**: UI updates in Preact, logic adjustments in Go, and prompt engineering in Python are reflected immediately without necessitating a full process restart.
- **Schema Lifecycle**: Modifications to the `uca.yaml` manifest trigger automated `tidy` operations to reconcile database collections and regenerate dependency locks.

---

## 4. Security Architecture: Integrity Sandboxing & Least Privilege

Uca adheres to a "Security-by-Design" philosophy to mitigate risks associated with prompt injection and accidental state corruption.

- **Read-Only Defaults**: By default, AI agents are restricted to **idempotent GET operations**. Agents cannot mutate service state or persistence records without explicit structural authorization.
- **Authorized Mutation**: Write capabilities must be explicitly granted by the developer via "Write-Action" hooks defined within the `uca.yaml` registry.
- **Secret Isolation**: Sensitive environment variables are defined in the manifest but persisted in a VCS-ignored `.uca/.env` file. These are injected into the runtime root, maintaining strict separation between configuration and sensitive data.

---

## 5. Deployment: Immutable Artifact Architecture

The `uca export` command generates a production-ready, **OCI-compliant "Appliance"**.

- **Encapsulated Payload**: The appliance bundles the compiled Go supervisor, the hermetic Python runtime environment, optimized Preact assets, and the unified interop registry.
- **High-Density Compression**: Artifacts are exported as `.tar.zst` archives. Utilizing **Zstandard (zstd)** ensures minimal storage footprints and significantly accelerated deployment cycles compared to traditional container layers.
- **Absolute Hermeticity**: All language runtimes and system dependencies are self-contained within the appliance's `.uca/` directory, eliminating environmental drift and ensuring consistent execution across diverse infrastructure.
