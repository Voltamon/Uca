

# Architectural Blueprint: Uca Microframework Specification

The Uca microframework is engineered as a high-performance, polyglot development environment. Its architecture is underpinned by four specialized foundational technologies:

1.  **PocketBase**: Persistent storage, identity management, and binary object hosting (Golang).
2.  **Preact**: High-efficiency UI orchestration and reactive component lifecycle (TypeScript).
3.  **PicoCSS**: Low-footprint, semantic design system for rapid interface layout (TypeScript/CSS).
4.  **SmolAgents**: AI agent execution environment with extensible tool-calling abstractions (Python).

---

## 1. Architectural Topology: The Hermetic Environment

Uca enforces a strict **discrete runtime separation** (language-per-component) to maximize consistency, maintainability, and execution performance.

| Component Layer | Core Framework | Implementation Language |
| :--- | :--- | :--- |
| **Intelligent Agents** | SmolAgents | Python |
| **Presentation (Views)** | Preact + PicoCSS | TypeScript (`.tsx`) |
| **Business Logic** | PocketBase (Embedded) | Golang |

### Workspace Directory Structure

| Path            | Significance  | Functional Purpose                                                      |
| :-------------- | :------------ | :---------------------------------------------------------------------- |
| **`.uca/`**     | **Mandatory** | Polyglot interop layer, hermetic runtimes, and local environment state. |
| **`agents/`**   | Optional      | Source definitions for AI agents (restricted to `.py`).                 |
| **`assets/`**   | Optional      | Static binary resources and immutable media artifacts.                  |
| **`modules/`**  | Optional      | Cross-runtime reusable logic modules (supporting `.py`, `.go`, `.ts`).  |
| **`pages/`**    | Optional      | View-layer component landscape (standardized on `.tsx`).                |
| **`services/`** | Optional      | Golang-based backend services and PocketBase extensions.                |
| **`deps.lock`** | **Mandatory** | Read-only dependency resolution manifest for version parity.            |
| **`uca.yaml`**  | **Mandatory** | Declarative application manifest and configuration source of truth.     |

## 2. Manifest Specification: `uca.yaml`

Uca utilizes a declarative paradigm where the `uca.yaml` manifest serves as the primary system configuration orchestrator.

### Primary Schema Definitions

#### `app` (Mandatory)
- `name`: (Mandatory) Unique identifier; serves as the project's root namespace and parent directory.
- `version`: (Optional) Semantic versioning (Defaults to `v1.0`).
- `port`: (Optional) Multi-service port mapping (Sub-tags: `frontend:`, `backend:`, `ai:`).
- `icon`: (Optional) Path to project branding (e.g., `.ico` or `.png` within `assets/`).
- `key`: (Optional) Enumeration of required environment variable keys for runtime injection.

#### `pages` (Optional)
*Governs frontend orchestration and automated asset generation.*
- `name`: (Mandatory) Generates `<name>.tsx` in `pages/`, exposed via the `/<name>` route.
- `route`: (Optional) Explicit URI override (e.g., `/<route>`).
- `role`: (Optional) RBAC (Role-Based Access Control) constraint; must be registered in the Uca context.
- `render`: (Optional) Rendering strategy: `ssg` (Static Site Generation), `spa` (Single-Page App, default), or `ssr` (Server-Side Rendering).
- `loader`: (Optional) Backend reference to a service function; returns JSON payload as component props.

#### `services` (Optional)
*Governs backend service synthesis and PocketBase integration.*
- `name`: (Mandatory) Generates `<name>.go` in `services/`, exposed via `/api/<name>/*`.
- `desc`: (Optional) Metadata description; required if registering methods as AI-invocable tools.
- `methods`: (Mandatory) Whitelist of permitted HTTP verbs (GET, POST, etc.).
- `schema`: (Optional) Declarative database collection definition; auto-generates CRUD infrastructure.
- `actions`: (Optional) Explicit Go function hooks registered within the Uca interop map.

#### `agents` (Optional)
*Governs AI agent synthesis and runtime orchestration.*
- `name`: (Mandatory) Generates `<name>.py` in `agents/`, exposed via `/api/chat/<name>`.
- `model`: (Mandatory) LLM provider/model identifier (e.g., `"github/gpt-4o"`).
- `tools`: (Optional) Reference list of authorized service methods or standard SmolAgents tools.
- `context`: (Optional) Service function reference for dynamic JSON context injection.
- `timeout`: (Optional) Execution lifecycle duration (Default: 30s).

## 3. CLI Orchestration: Lifecycle State Machine

The Uca CLI acts as the authoritative orchestrator, performing real-time static analysis and type-safety verification during execution.

### Core Lifecycle Commands
- `init <app-name>`: Scaffolds the hermetic project structure and base manifest.
- `dev`: Initializes the concurrent development runtime with multi-layer Hot Module Reloading (HMR).
- `tidy`: The reconciliation engine; synchronizes `uca.yaml`, executes migrations, and reconciles `deps.lock`.
- `test`: Executes the unified test suite across all runtime layers.
- `export <binary-name>`: Builds an immutable containerized appliance in `.tar.zst` format.

### Hierarchical Administrative Interface

The CLI utilizes a granular command hierarchy for precise component management:

| Functional Domain | Management Commands |
| :--- | :--- |
| **System Config** | `icon <path>`, `port <name> <val>`, `version` |
| **View Layer** | `add`, `remove`, `deps`, `test`, `list`, `info` |
| **Logic Services** | `add <name> <methods>`, `remove`, `deps`, `test`, `list`, `info` |
| **Agent Runtimes** | `add <name> <model>`, `remove`, `deps`, `test`, `list`, `info` |
| **Persistence** | `list`, `schema`, `seed <file>`, `reset <table>` |
| **Authorization** | `add`, `remove`, `list` |
| **Environment** | `add <name> <value>`, `remove`, `info` |
| **Observability** | `logs {list\|info\|clear}`, `tests {add\|remove\|list}` |

## 4. Runtime Integrity & Security Architecture

- **Static Type Parity**: Automated synthesis of Golang constructs into TypeScript interfaces to ensure end-to-end type safety.
- **IPC Isolation**: Inter-process communication is localized to a high-speed HTTP bus bound to `localhost`.
- **Centralized Registry**: A unified interop map for deterministic resolution of loaders and tools.
- **Secret Management**: Sensitive data is isolated in `.uca/.env`, excluded from version control, and injected at the runtime root.
- **Principal of Least Privilege**: Agents operate in a **Read-Only** sandbox by default. Data mutation requires explicit authorization via "Write-Action" hooks to mitigate injection risks.
- **Fault Tolerance**: The Go-based supervisor continually monitors child processes (Python Agents), implementing an automated restart loop upon failure.
- **Deployment Efficiency**: Artifacts utilize Zstandard (`zstd`) compression to ensure mission-critical small footprints and near-instant load times.
